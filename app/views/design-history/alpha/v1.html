Sprint 1: initial prototype

Ideation

We held a two-part ideation session for user needs in compliance. The result was a set of features that would have the most impact and that were technically feasible.

  • Searching for monitoring stations by location
  • Searching for monitoring stations by local authority
  • Searching for monitoring stations across multiple networks at once
  • Specific search options such as ‘eastings and northings’
  • Identify each monitoring station within a location or local authority
  • To be able to search within ‘x’ miles of a location
  • Searching and filtering by pollutant
  • Filtering by different formats (undefined)
  • Visual representations of measured data with selectable time frames
  • Warnings or alerts for missing data or data anomalies
  • Adding a local authority boundary layer to the interactive map
  • Data standardisation in structure and format
  • Consistent categorisations across all monitoring sites
  • Consistent naming conventions

Design hypothesis

We’ve seen users using the map on UK AIR to find monitoring stations.

We think this is because there’s no easy way to search for a location or filter monitoring stations without using a map. The data selector that could do this task is unintuitive and frustrating to use.

If we design an easy and simple way for users to find monitoring stations by location, users will find the data they need more efficiently, without relying on vision or digital skills.

The focus

We could not tackle all these user needs in one prototype. We decided to focus on the following core user needs for finding and identifying the right location and monitoring station data to download:

  • to search by location, to find a specific monitoring station or the station coverage for an area or local authority
  • to search for stations by a specific pollutant within a larger area
  • to filter by pollutant
  • to view pollutant coverage for a single monitoring station

Testing this prototype with users was not about usability. It was about exploring the presentation and format of information we know users need. We wanted to find out:

  • if the journey had the right steps
  • if users needed any other information or control of segmentation of data
  • what else we needed to consider to solve user needs for choosing and getting monitoring station data

Before you start

This service uses data from the:

  • Automatic Urban and Rural network (AURN)
  • locally managed automatic monitoring network

The AURN provides the highest quality air pollution data in the UK. You can use AURN data to check compliance with the UK Air Quality Standards Regulations 2010.

Most locally managed automatic monitoring stations are run by local authorities. You should not use data from this network to check compliance with the UK Air Quality Standards Regulations 2010.

This service does not:

  • check local air quality
  • do air pollution assessments
